## ナレーション台本 (約2分・句点なし)

**[S1 表紙 / 10秒]**
東京レイルコンフォートです  東京の電車の「空いている時間」を、オープンデータから推定して選べるようにする Web アプリを作りました

**[S2 課題 / 20秒]**
毎朝の通勤電車の混雑は、都民の生活の質と生産性を下げ続けています  でも私たちは「今日、どの区間が、何時なら空いているか」を事前に比べて行動を変える手段を持っていません  混雑を避けたくても、選ぶための情報が無いんです

**[S3 壁 / 20秒]**
ここで大きな壁にぶつかりました  リアルタイムの車両ごとの混雑、その「実測値」は、東京の電車のオープンデータには存在しないんです  ODPTが出しているのは運行情報だけ、JRのリアルタイムデータにも混雑の項目はありません

**[S4 新規性 / 25秒]**
そこで発想を変えました  実測が無いなら、推定する  国土交通省の混雑率という統計を土台に、時刻表から時間帯ごとのカーブを引き、リアルタイムの運行情報の遅延で補正します  複数のオープンデータを掛け合わせることで、「実測が無い」という制約を乗り越えるのがこの作品の新しさです

**[S5 デモ / 25秒]**
実際の画面です  路線を選ぶと、区間ごと・時間帯ごとの推定混雑が色で並びます  そして「この区間、空いているのは何時か」を提案します  たとえばこの区間、8時台は激しく混みますが、10時台なら快適です  全ての数値に「これは推定です」と明示し、データの出典と生成時刻も必ず表示しています

**[S6 データ / 10秒]**
使っているのは、ODPTの運行情報と時刻表、国土交通省の混雑率調査、駅別乗降人員  すべて公開されているオープンデータです

**[S7 インパクト+紹介 / 10秒]**
オフピーク移動の後押しは、国が進める政策とも方向が一致し、都民の手取り時間を増やします  個人で企画から実装、公開まで手がけました  ありがとうございました
