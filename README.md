# 東京レイルコンフォート（仮）

東京の電車の混雑を「統計 × 時刻表」で時間帯別に**推定**し、リアルタイムの運行情報（遅延）で**補正**して可視化。空いている時間・区間を提案し、オフピーク移動を後押しする Web アプリ。
都知事杯オープンデータ・ハッカソン2026 個人参加作品。

## これは何で、何でないか
- ✅ 統計と時刻表から混雑を「推定」し、運行情報で補正して見せる
- ❌ リアルタイムの車両別混雑の「実測」表示ではない（東京の電車にそのオープンデータは存在しない）

## Getting started
```bash
cp .env.example .env.local   # 値を埋める（ODPT consumerKey / Supabase）
pnpm install                 # or npm install
pnpm dev
```

## 開発の進め方
- プロジェクトの前提・制約・規約は `CLAUDE.md`
- 構想・データ根拠は `docs/CONCEPT.md`
- 実装マイルストーンは `TASKS.md`（上から順に）
- DBスキーマは `supabase/schema.sql`

## 事前準備
1. https://developer.odpt.org/ で登録し `consumerKey` を取得
2. 国交省「都市鉄道の混雑率調査」最新年度の資料2 PDF を `data/` に配置
3. Supabase プロジェクトを作成し、`supabase/schema.sql` を適用

## データ出典
- 公共交通オープンデータセンター（ODPT）
- 国土交通省 都市鉄道の混雑率調査
- 各鉄道事業者 駅別乗降人員 / 東京都オープンデータカタログ

本アプリは公共交通オープンデータセンターのデータを利用しています。
